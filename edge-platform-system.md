[최종안] 엣지 AI 기반 지능형 이슈 관리 및 자동 보고 시스템
본 문서는 ASUS Ascent GX10, Raspberry Pi 5, Jetson Nano, Synology NAS를 결합하여 다수의 사원이 외부에서 사용 가능한 하이브리드 AI 자동화 환경 구축 전략을 다룹니다.
1. 하드웨어 인프라 구성 (4-Node Cluster)
각 장비의 특성에 맞춰 역할을 분산하여 시스템 안정성과 효율성을 극대화합니다.
| 장비 | 역할 | 핵심 서비스 |
|---|---|---|
| Raspberry Pi 5 | 통합 게이트웨이 | n8n (워크플로우), Cloudflare Tunnel, Nginx Proxy Manager |
| ASUS GX10 | 고성능 AI 엔진 | 로컬 LLM (GLM-4.7), Ollama, LangGraph 연산 |
| Synology NAS | 데이터 저장 및 관리 | Gitea (소스), Redmine (이슈), PostgreSQL (DB) |
| Jetson Nano | 검증 및 CI/CD | 코드 빌드 테스트, 엣지 환경 시뮬레이션 |
2. 핵심 서비스 워크플로우
① 이슈 등록 (Input Phase)
 * 채널: 사원 전용 n8n 웹 폼 (외부 접속 시 Cloudflare Zero Trust 인증 필수)
 * 프로세스: 사원이 정해진 양식에 맞춰 이슈(제목, 내용, 카테고리 등)를 입력하면 n8n이 이를 수집하여 즉시 분석 단계로 전달합니다.
② 지능형 분석 (Processing Phase)
 * AI 라우팅: ASUS GX10에서 구동되는 AI가 내용을 분석하여 카테고리를 분류합니다.
 * 이슈 생성: 분석된 데이터는 자동으로 NAS의 Redmine에 신규 티켓으로 생성됩니다.
 * 코드 검증: 필요한 경우 Jetson Nano가 관련 코드를 빌드하여 결과값을 리포트에 추가합니다.
③ 보고서 발송 (Output Phase)
 * 개인별 맞춤 발송: n8n의 조건부 로직(Switch)을 통해 AI가 판별한 담당자에게만 이메일/메신저로 보고서를 전송합니다.
 * 아카이빙: 생성된 모든 보고서와 분석 결과는 Synology NAS의 지정된 경로에 자동 저장됩니다.
3. 구축 및 운영 전략 (보안/효율)
 * 보안: 전용 도메인과 Cloudflare Tunnel을 연결하여 포트 개방 없이 안전한 외부 접속 환경을 구축합니다.
 * 전력 효율: 24시간 가동되는 Pi 5가 대기하다가, 고부하 작업 시에만 ASUS GX10에 API 요청을 보내는 방식으로 운영 비용을 절감합니다.
 * 협업 인프라: Gitea와 Redmine을 통해 모든 작업 이력을 투명하게 관리하고 AI가 이를 학습 데이터로 활용하게 합니다.
4. 시스템 도입의 장단점
✅ 장점 (Pros)
 * 압도적인 보안 및 로컬 성능: ASUS GX10의 Blackwell 기반 가속력을 통해 사내의 민감한 데이터를 외부 유출 없이 로컬에서 완벽하게 처리할 수 있습니다.
 * 안정적인 분산 구조: 각 장비가 역할을 분담하여 특정 장비에 장애가 발생해도 데이터 저장(NAS)이나 전체 흐름 제어(Pi 5) 기능이 마비되지 않습니다.
 * 지능형 자동 분류: 다수의 직원이 보낸 데이터를 AI가 스스로 분류하고 적합한 담당자에게만 전달하므로 관리자의 리소스를 획기적으로 줄여줍니다.
❌ 단점 (Cons)
 * 초기 설정 복잡도: 4대 장비 간의 통신 보안(SSL/TLS), API 연동, Docker 네트워크 설정 등 초기 인프라 구축에 높은 기술적 이해도가 요구됩니다.
 * 유지보수 관리 포인트: 4대의 OS와 수십 개의 컨테이너를 개별적으로 관리 및 업데이트해야 하는 운영 부담이 존재합니다.
 * 내부 네트워크 의존성: 기기간 대용량 데이터 전송(모델 로딩, 로그 수집 등)이 잦으므로 고성능 스위칭 허브와 기가비트 이상의 네트워크 환경이 필수적입니다.
5. 실행 체크리스트 (Action Plan)
 * [ ] 도메인 구매 및 Cloudflare 설정: 외부 접속 통로 확보
 * [ ] NAS 서비스 안정화: Gitea 및 Redmine 설치 및 외부 API 허용 설정
 * [ ] n8n 워크플로우 설계: 이슈 수집 폼 및 조건부 발송(Switch) 로직 구현
 * [ ] AI 서빙 설정: ASUS GX10에 Ollama 및 GLM-4 모델 서빙 및 API 엔드포인트 생성
