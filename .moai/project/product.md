## 제품 설명

vibe-coding-mcp는 Claude Code 기반의 다중 에이전트 코딩 자동화 시스템입니다. MoAI-ADK(Multi-Agent Orchestrated AI Development Kit) 프레임워크를 구현하며, Domain-Driven Design(DDD) 방법론과 SPEC-First 개발 워크플로우를 통해 고품질 소프트웨어 개발을 자동화합니다.

### 핵심 가치 제안

**품질 우선 개발**: TRUST 5 프레임워크를 통한 자동화된 품질 게이트로 테스트 가능성, 가독성, 통일성, 보안, 추적가능성을 보장합니다.

**지능형 작업 분해**: 복잡한 개발 작업을 자동으로 분해하고 최대 10개의 전문화된 에이전트에게 병렬로 위임하여 개발 속도를 획기적으로 향상시킵니다.

**토큰 최적화**: 3단계 점진적 공개 시스템으로 초기 토큰 로드를 67% 이상 감소시키고, 효율적인 컨텍스트 관리로 200K 토큰 예산 내에서 대규모 프로젝트를 처리합니다.

**SPEC-First DDD**: 요구사항 명세(SPEC)부터 시작하여 설계(Strategy), 구현(DDD), 문서화(Docs)까지 이어지는 구조화된 개발 워크플로우를 제공합니다.

## 타겟 사용자

### 1. 소프트웨어 개발자

- **복잡한 시스템 개발자**: 마이크로서비스, 분산 시스템, 대규모 애플리케이션을 개발하는 개발자
- **AI 도구 활용 개발자**: Claude, GitHub Copilot 등 AI 도구를 활용하여 생산성을 극대화하려는 개발자
- **품질 중심 개발자**: 자동화된 품질 검증과 리팩토링을 통해 지속 가능한 코드를 작성하는 개발자

### 2. AI 자동화 팀

- **DevOps 엔지니어**: CI/CD 파이프라인에 AI 기반 코드 생성 및 검증을 통합하려는 팀
- **플랫폼 엔지니어**: 개발 플랫폼에 AI 에이전트 시스템을 구축하여 팀 생산성을 향상시키려는 엔지니어
- **기술 리드**: 팀의 개발 프로세스를 표준화하고 자동화하는 방법을 찾는 리드 개발자

### 3. 스타트업 및 소규모 팀

- **빠른 프로토타이핑**: 아이디어를 빠르게 검증하고 MVP를 개발해야 하는 스타트업
- **리소스 제약 환경**: 제한된 인력으로 고품질 소프트웨어를 개발해야 하는 소규모 팀
- **기술 부채 관리**: 자동화된 리팩토링과 테스트로 기술 부채를 체계적으로 관리해야 하는 팀

## 주요 기능

### 1. 다중 에이전트 시스템 (19개 에이전트)

**Manager 에이전트 (7개)**
- `manager-spec`: EARS 형식의 요구사항 명세 생성
- `manager-ddd`: DDD 기반 구현 (ANALYZE-PRESERVE-IMPROVE)
- `manager-docs`: Nextra 기반 문서 자동 생성
- `manager-quality`: TRUST 5 품질 검증
- `manager-project`: 프로젝트 구성 및 구조 관리
- `manager-strategy`: 시스템 설계 및 아키텍처 결정
- `manager-git`: Git 작업 및 브랜칭 전략

**Expert 에이전트 (8개)**
- `expert-backend`: API 개발, 서버 측 로직
- `expert-frontend`: React 컴포넌트, UI 구현
- `expert-security`: 보안 분석, 취약점 평가
- `expert-devops`: CI/CD 파이프라인, 인프라
- `expert-performance`: 성능 최적화, 프로파일링
- `expert-debug`: 디버깅, 오류 분석
- `expert-testing`: 테스트 생성, 전략 수립
- `expert-refactoring`: 코드 리팩토링, 아키텍처 개선

**Builder 에이전트 (4개)**
- `builder-agent`: 새로운 에이전트 정의 생성
- `builder-command`: 슬래시 명령 생성
- `builder-skill`: 스킬 모듈 생성
- `builder-plugin`: 플러그인 생성

### 2. SPEC-First DDD 워크플로우

**1단계: SPEC (`/moai:1-plan`)**
- EARS(Easy Approach to Requirements Syntax) 형식으로 요구사항 명세화
- manager-spec 에이전트가 구조화된 요구사항 문서 생성
- .moai/specs/SPEC-XXX/spec.md에 명세 저장

**2단계: DDD (`/moai:2-run`)**
- ANALYZE: 요구사항 분석 및 설계
- PRESERVE: 기존 동작 보존을 위한 특성화 테스트
- IMPROVE: 점진적 기능 개선 및 구현
- 85% 이상 테스트 커버리지 목표

**3단계: Docs (`/moai:3-sync`)**
- Nextra 기반 문서 사이트 자동 생성
- API 문서, 아키텍처 다이어그램, 사용자 가이드
- 마크다운/Mermaid 다이어그램 통합

### 3. 점진적 공개 (Progressive Disclosure)

**3단계 스킬 로딩 시스템**
- **레벨 1**: 메타데이터만 로드 (스킬당 ~100 토큰)
- **레벨 2**: 스킬 본문 로드 (트리거 시 ~5K 토큰)
- **레벨 3**: 참조 파일 온디맨드 로딩

**토큰 절감 효과**
- 초기 토큰 로드 67% 이상 감소
- 90K → 600 토큰 (manager-spec의 경우)
- 필요할 때만 전체 스킬 콘텐츠 로드

### 4. TRUST 5 품질 프레임워크

**5대 핵심 기둥**
1. **Tested**: 85% 커버리지, 특성화 테스트, 명세 기반 테스트
2. **Readable**: 명확한 명명 규칙, Ruff 린터, 일관된 스타일
3. **Unified**: Black 포매터, isort 임포트 정렬, 일관된 포맷
4. **Secured**: OWASP 준수, 보안 에이전트 분석, 취약점 스캔
5. **Trackable**: 구조화된 커밋 메시지, Git 전략, 변경 추적

### 5. MCP(Model Context Protocol) 통합

**Context7 서버**
- 공식 라이브러리 문서 온디맨드 로딩
- 최신 베스트 프랙티스 실시간 검색
- 품질 검증을 위한 최신 표준 확인

**Sequential Thinking 서버**
- 복잡한 문제의 단계별 분해
- 추론 과정 추적 및 수정
- 아키텍처 결정 지원

### 6. 자동화된 도구 통합

**AST-Grep**
- 구조적 코드 패턴 매칭
- 정확한 리팩토링 candiitate 식별
- 오탐 방지 및 성능 최적화

**Git Workflow**
- 브랜칭 전략 자동화
- 커밋 메시지 포맷 검증
- 머지 리퀘스트 템플릿

**다중 LLM 지원**
- Claude (주요): 고품질 코드 생성
- GLM (비용 효율): 대규모 작업 병렬 처리
- Gemini/Copilot (옵션): 추가 검증 및 아이디어 생성

## 주요 혜택

### 1. 개발 생산성 향상

**시간 절약**
- 자동화된 코드 생성: 40-60% 개발 시간 단축
- 병렬 작업 처리: 최대 10개 에이전트 동시 실행
- 빠른 피드백 루프: 실시간 품질 검증

**품질 보장**
- 자동화된 테스트 생성 및 실행
- 일관된 코드 스타일 및 포맷팅
- 보안 취약점 자동 스캔

### 2. 팀 협업 개선

**표준화된 프로세스**
- SPEC-First 개발로 요구사항 명확화
- DDD 방법론으로 도메인 주도 설계
- 일관된 코드 리뷰 기준

**지식 공유**
- 자동 생성되는 문서
- 아키텍처 다이어그램 시각화
- 베스트 프랙티스 템플릿

### 3. 기술 부채 감소

**리팩토링 자동화**
- 안전한 코드 변환 (동작 보존)
- 특성화 테스트로 회귀 방지
- 점진적 아키텍처 개선

**테스트 커버리지**
- 85% 목표 커버리지 자동화
- 명세 기반 테스트 생성
- 변이 테스트 지원

## 사용 사례

### 1. 신규 프로젝트 개발

**시나리오**: 스타트업이 새로운 SaaS 플랫폼 개발

**워크플로우**:
1. `/moai:0-project`로 프로젝트 초기화
2. `/moai:1-plan`으로 핵심 기능 명세화
3. `/moai:2-run`으로 백엔드 API 개발
4. `/moai:3-sync`으로 프론트엔드 UI 개발
5. `/moai:3-sync`으로 문서 자동 생성

**결과**: 2주 만에 MVP 배포 완료

### 2. 레거시 코드 현대화

**시나리오**: 5년 된 모놀리식 애플리케이션 마이크로서비스로 전환

**워크플로우**:
1. `manager-spec`으로 현재 상태 분석 및 명세화
2. `manager-ddd`로 점진적 마이크로서 추출
3. `expert-testing`으로 특성화 테스트 생성
4. `expert-refactoring`으로 안전한 리팩토링
5. `manager-quality`로 품질 검증

**결과**: 6개월 만에 무중단 마이그레이션 완료

### 3. API 개발 및 문서화

**시나리오**: RESTful API 개발 및 OpenAPI 문서 자동화

**워크플로우**:
1. `/moai:1-plan "사용자 인증 API 개발"`
2. `expert-backend`로 FastAPI 엔드포인트 구현
3. `expert-testing`으로 통합 테스트 작성
4. `manager-docs`로 OpenAPI 스펙 생성
5. `/moai:3-sync`으로 문서 사이트 배포

**결과**: 개발과 동시에 최신 문서 자동 유지

### 4. 보안 감사 및 취약점 수정

**시나리오**: OWASP Top 10 취약점 점검 및 수정

**워크플로우**:
1. `expert-security`로 보안 감사 수행
2. `manager-quality`로 TRUST 5 보안 기둥 검증
3. `expert-backend`로 취약점 수정
4. `expert-testing`으로 회귀 테스트
5. Git 보고서 자동 생성

**결과**: 95% 이상 취약점 해결
